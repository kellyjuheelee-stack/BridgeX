## 0. 배경과 핵심 원칙

현재 STEP 2(진단)·STEP 3(콜드 이메일)·STEP 4(후속 대응)는 "AI 자동화"로 표기돼 있으나 실제로는
규칙 기반이며, 세 도구가 서로 단절돼 있다("진단 한 번 받고 끝"). 이를 **하나의 여정**으로 꿰어
회원 만족 → **컨설팅 전환**을 끌어올린다.

**전환 원칙 A (모든 기능의 설계 기준):**
> AI는 가치를 증명하되, "혼자서는 못 하겠다"는 벽을 자연스럽게 드러낸다.
> 각 도구는 (1) 진짜 잘 해주고, 동시에 (2) 전문가가 필요한 지점에서 자연스럽게 끝나야 한다.

**수익 구조 전제:** 진단·로드맵은 **무료 훅**이다. 실제 매출은 **유료 회원 특전 + 전문가 컨설팅**에서
나온다. 따라서 무료 훅은 비용을 아끼고(규칙 기반), 생성형 AI 비용은 매출과 직접 연결되는 곳
(유료 특전·컨설팅)에 쓴다.

**엔진 전략 (하이브리드):** 뼈대·판정·진행률은 규칙 기반(무료·즉시·안정). 회원이 체감하는
"맞춤 문장"만 나중에 Claude API로 교체 — 생성 함수 한 개만 감싸면 되도록 설계한다.
초기(고객 베이스 없음)에는 규칙 기반 코멘트로 충분히 구체적으로 간다.

## 2. 모듈 1 — 수출 실행 로드맵 [상세 스펙]

### 2.1 회원 경험 흐름
1. 진단 제출 → 기존 점수 결과 **아래에 로드맵이 자동 생성**된다.
2. 마이페이지 상단에 상시 노출: `내 수출 로드맵 — 16단계 중 N단계 완료 · M단계는 전문가 필요`.
3. `🟢 셀프` 단계 클릭 → 해당 도구로 이동 (예: 콜드 아웃리치 → `email.html`).
4. `🔴 전문가` 단계 클릭 → **그 단계 맥락이 프리필된 컨설팅 신청** (예: "CPNP 등록 관련 상담").
5. 회원이 단계 완료를 셀프 체크 → 진행률↑, 추천 다음 액션 갱신.
6. **전문가 벽이 매 로그인마다 보이는 것**이 핵심 전환 장치.

### 2.2 로드맵 단계 모델 (16단계)

태그 범례: 🟢 셀프(AI/도구가 도움) · 🟡 셀프 점검 후 전문가 확정 · 🔴 전문가 필요(컨설팅)

| # | 단계 | 태그 | 연결 | 완료/필요 판정 근거(진단 데이터) |
|---|------|------|------|------|
| **A. 기반 — 제품·규제** ||||
| 1 | 국내 판매 실적·인증 정리 | 🟢 | 진단 자동 판정 | `isSellingInKorea`∈{판매 중,테스트 판매 중} && 실인증 보유 → 완료 |
| 2 | EU 책임자(RP) 지정 | 🔴 | 컨설팅 | `euComplianceReadiness` 에 'EU 책임자(Responsible Person) 지정' 포함 → 완료 |
| 3 | CPNP 사전 등록 | 🔴 | 컨설팅 | 'CPNP 사전 등록' 포함 → 완료 |
| 4 | PIF(제품정보파일) 구비 | 🔴 | 컨설팅 | '제품정보파일(PIF) 구비' 포함 → 완료 |
| 5 | CPSR(안전성보고서) | 🔴 | 컨설팅 | '제품안전성보고서(CPSR) 작성' 포함 → 완료 |
| 6 | INCI 전성분 EU 적합성 | 🟡 | 컨설팅 | '전성분(INCI) EU 규정 적합성 확인' 포함 → 완료 |
| 7 | EU 라벨링·알레르겐 표시 | 🔴 | 컨설팅 | 'EU 라벨링 요건 충족' && '향료 알레르겐(80종) 표시 대응' 둘 다 → 완료 |
| 8 | PPWR 포장 대응 | 🔴 | 컨설팅 | `packagingReadiness` 에 준비 항목(DoC·재활용 등) 보유 수로 판정 |
| **B. 영업 자료** ||||
| 9 | AI 카탈로그·회사소개 생성 | 🟢 | **모듈 2** | 브로셔 없음(자료 미보유/painPoint) → 필요 |
| 9b | 기존 카탈로그 업로드+바이어레디 진단 | 🟡→🔴 | **모듈 2** | 자료 보유 시 노출 |
| 10 | Offer Sheet(가격·MOQ·Incoterms) | 🟡 | 모듈 2 | 항상 권장 |
| **C. 바이어** ||||
| 11 | 타겟 바이어 리스트 | 🔴 | 컨설팅(코어 자산) | 항상 전문가 — AI 자동발굴 안 함(할루시네이션 위험) |
| 12 | 콜드 아웃리치 이메일 | 🟢 | `email.html` | 도구로 수행 |
| 13 | 박람회 후속·리마인더 | 🟢 | `email.html` | 도구로 수행 |
| 14 | 관심 답장 대응 | 🟢 | `email.html` | 도구로 수행 |
| 15 | 미팅·협상 | 🔴 | 컨설팅 | 항상 전문가 |
| 16 | 계약 | 🔴 | 컨설팅 | 항상 전문가 |

> 셀프 단계는 아웃리치(12~14)와 카탈로그 생성(9)에 몰려 있어 회원이 직접 하며 만족을 얻고,
> 벽은 규제(2~8)·바이어 매칭(11)·협상/계약(15~16)에 집중된다 — 정확히 원칙 A.
>
> 참고: 9번(생성)과 9b번(업로드+진단)은 회원의 브로셔 유무에 따라 **둘 중 하나만** 표시되므로
> 회원이 실제 보는 단계 수는 16으로 유지된다.

### 2.3 단계 상태 계산
각 단계는 두 층으로 상태가 결정된다.

- **파생 상태(derived):** 진단 데이터로 규칙 산출 → `done | needed | expert`.
- **회원 셀프체크(memberCheck):** 회원이 완료 토글하면 우선 적용(override).
- **표시 상태 = memberCheck 있으면 그 값, 없으면 derived.**

진행률 = (표시 상태가 `done` 인 단계 수) / 16.
"전문가 필요 개수" = 표시 상태가 `expert`(미완료)인 🔴/🟡 단계 수 → 상단 배지에 노출.

### 2.4 백엔드
- 신규 서비스 `backend/src/services/roadmap.service.js`
  - `generateRoadmap(diagnosisRequest, memberProgress)` → 16단계 배열 반환
    (각: id, phase, title, tag, link, derivedStatus, displayStatus, comment).
  - 코멘트는 규칙 기반 문자열(초기). **나중에 이 함수 내부만 Claude 호출로 교체.**
- 저장: 회원 셀프체크 상태를 JSON으로 보관.
  - Prisma `Member` 에 `roadmapProgress String?` (JSON 문자열: `{ "<stepId>": { "done": true, "doneAt": "..." } }`).
  - (대안: 별도 `RoadmapProgress` 테이블. v1은 JSON 필드로 단순화.)
- API
  - `GET  /api/members/me/roadmap` → 최신 진단 기반 로드맵 + 진행률.
  - `POST /api/members/me/roadmap/steps/:stepId/toggle` → 완료/해제 저장.
  - 컨설팅 프리필: 기존 컨설팅/상담 신청에 `stepContext`(단계명·설명) 전달 → 관리자 리드에 표시.

### 2.5 프론트
- `mypage.html` 에 **로드맵 카드 섹션** 추가(기존 결과 카드 아래). 별도 페이지 아님.
  - 상단 진행률 바 + "N/16 완료 · M단계 전문가 필요" 배지.
  - 3구간(기반/영업자료/바이어) 아코디언 또는 리스트, 각 단계 태그 뱃지·체크박스·연결 버튼.
  - 🟢 → 도구 링크, 🔴/🟡 → 컨설팅 신청(맥락 프리필).
- 진단이 없는 회원에겐 "진단부터 받기" CTA.

### 2.6 v1 범위
- 포함: 로드맵 생성·표시·셀프체크·진행률·셀프/전문가 연결·컨설팅 맥락 프리필.
- 제외(나중): Claude 문장 생성, 활동 자동감지 진행, 재진단 재생성, 카탈로그 생성기(모듈 2).

## 3. 모듈 2 — AI 브랜드 카탈로그 [개요, 다음 스펙]

### 3.1 목적과 두 갈래
- **브로셔 없는 영세 회원:** 로고+제품+회사소개 입력 → 브랜드 톤&매너에 맞춘
  **온라인 카탈로그(공유 링크형 미니 페이지) + 영문 회사소개** 자동 생성 → 마이페이지 자산.
  가치 = **디자인·대행 비용 절감**을 즉시 체감. → **유료 회원 특전.**
- **이미 카탈로그가 있는 회원:** 업로드 → BridgeX가 아래 가치를 준다(단순 보관 아님).

### 3.2 "이미 카탈로그가 있는 회원"에게 주는 VALUE (무료 개방 — 벽 노출용)
1. **🔴 EU 클레임 컴플라이언스 점검(킬러):** 한국 카탈로그에 흔한 EU 금지·제한 표현
   (미백/화이트닝, 주름 개선, 항노화, 의학적·치료 뉘앙스, 무독성 등)을 스캔해 플래그.
   EU 화장품 규정(EC 1223/2009) + 클레임 공통기준(Reg 655/2013) 위반 위험 → **컨설팅 벽.**
2. **바이어레디 갭 진단:** EU B2B 바이어 기대 요소(INCI, 인증 vegan/cruelty-free/GMP,
   지속가능성·PPWR, MOQ·Incoterms·shelf life) 대비 누락 항목 진단.
3. **영문·B2B 리포맷 제안:** 소비자용 카탈로그 → 바이어용 영문 Offer Sheet 형태 재구성.
4. **자산 라이브러리:** 마이페이지 보관, 아웃리치 이메일(STEP 3)에 링크·첨부.

→ 없는 회원은 만들어주고(비용 절감·유료), 있는 회원은 점검해주고(벽 노출·무료→컨설팅). 둘 다 전환.

### 3.3 무료 회원 티저 — 프로급 샘플 갤러리
- 유료 생성 전, **실제 쓰이는 프로페셔널 카탈로그 톤의 샘플 2~3종** 노출
  (예: 스킨케어 브랜드형 / 색조 브랜드형 / 미니멀 클린뷰티형).
- "이 회원의 로고·제품으로 바꾸면 이렇게 나옵니다"는 before/after 배치로 생성 욕구 → 유료 전환.

### 3.4 형식·엔진 방향(스펙에서 확정)
- "온라인 카탈로그"는 PDF보다 **공유 링크형 미니 페이지** 권장(링크 전송·열람 추적, STEP 3·4 연계).
- 생성 엔진: 카피·톤&매너 = 생성형 AI(Claude 등) + 레이아웃 = 템플릿. 로고 업로드 처리.
- 업로드 진단(3.2): 문서 파싱 + 클레임 규칙/AI 스캔.

### 3.5 의존성·가정
- "유료 회원" 게이팅은 **결제/멤버십 등급 시스템이 아직 없다면**, 초기엔 "문의/대기" 플래그로
  막고 결제 시스템은 **별도 과제**로 둔다. (모듈 2 스펙에서 재확정)

## 4. 결정 로그 (이번 브레인스토밍)
1. 전환 순간 = **벽에 부딪힐 때** (원칙 A).
2. 세 도구를 잇는 **통합 워크플로우**를 추가한다.
3. 중심 축 = **지속되는 수출 실행 로드맵**.
4. 엔진 = **하이브리드**, 초기엔 규칙 기반(무료 서비스 비용 규율).
5. 진행 방식 = **회원 셀프체크 + 도구 연결**.
6. 영업자료 확장 = **AI 카탈로그 생성(유료 특전) + 기존 카탈로그 업로드·진단(무료)**.
7. 무료 회원에겐 **프로급 카탈로그 샘플 갤러리**로 티저.
8. 구조 = **모듈 1(로드맵) 먼저 구현, 모듈 2(카탈로그) 별도 스펙**.

## 5. 열린 질문 (모듈 2 스펙에서 다룸)
- 결제/멤버십 등급 시스템 존재 여부와 게이팅 방식.
- 카탈로그 생성 엔진 세부(레이아웃 템플릿 수, 로고/이미지 처리, 저장 포맷).
- 업로드 문서 파싱 범위(PDF/이미지/한글 파일).
