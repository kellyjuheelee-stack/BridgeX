# 구글 로그인 켜기 — 대표님이 브라우저에서 할 일 (아주 쉬운 설명)

> 코드는 이미 다 만들어서 웹사이트에 올렸어요. 이제 **구글이랑 우리 서버를 연결하는 열쇠**만
> 대표님이 직접 만들어서 끼워주면 "Google로 계속하기" 버튼이 진짜로 작동해요.
> 아래를 **위에서부터 순서대로** 따라 하면 됩니다. 크게 **3단계**예요.

- 걸리는 시간: 약 15~20분
- 준비물: 인터넷 되는 컴퓨터, 구글 계정(회사 지메일 권장)
- 자주 쓰는 값 2개 (복사해서 쓰세요):
  - **A값 (구글에 넣을 주소):** `https://lomjryokqskruzdhieol.supabase.co/auth/v1/callback`
  - **B값 (우리 사이트 주소):** `https://bridgexkorea.com/auth/callback`

## 1단계. 구글에서 "열쇠" 만들기 (Google Cloud Console)

구글이 "이 웹사이트는 믿을 수 있어요"라고 인정해주는 **아이디(Client ID)** 와
**비밀번호(Client Secret)** 를 발급받는 단계예요.

### 1-1. 사이트 접속 & 프로젝트 만들기
1. 브라우저에서 **https://console.cloud.google.com** 접속 → 회사 구글 계정으로 로그인.
2. 화면 맨 위 왼쪽에 있는 **프로젝트 선택** 칸(회사이름▼ 처럼 생김)을 클릭.
3. 오른쪽 위 **새 프로젝트** 클릭 → 이름에 `BridgeX` 입력 → **만들기**.
4. 잠깐 기다린 뒤, 다시 위쪽 프로젝트 칸을 눌러 방금 만든 **BridgeX** 를 선택.
   (앞으로 모든 작업은 이 BridgeX 프로젝트 안에서 해야 해요.)

### 1-2. 동의 화면 설정 (사용자에게 뭐라고 보여줄지)
1. 왼쪽 위 **☰ 메뉴** → **API 및 서비스** → **OAuth 동의 화면** 클릭.
2. User Type(사용자 유형)에서 **외부(External)** 선택 → **만들기**.
3. 아래 칸을 채워요:
   - **앱 이름:** `BridgeX`
   - **사용자 지원 이메일:** 대표님 회사 이메일 선택
   - **개발자 연락처 이메일:** 같은 이메일 입력
   - (승인된 도메인 칸이 있으면 `bridgexkorea.com` 입력)
4. **저장 후 계속**을 계속 눌러 끝까지 넘어가요. (범위·테스트 사용자는 그냥 넘겨도 됩니다.)

### 1-3. 진짜 열쇠(Client ID) 발급
1. 왼쪽 메뉴에서 **사용자 인증 정보(Credentials)** 클릭.
2. 위쪽 **+ 사용자 인증 정보 만들기** → **OAuth 클라이언트 ID** 선택.
3. **애플리케이션 유형:** `웹 애플리케이션` 선택.
4. **이름:** `BridgeX Web` (아무거나 괜찮아요).
5. 아래로 내려서 **승인된 리디렉션 URI** 부분 → **+ URI 추가** 클릭 →
   위에서 복사해둔 **A값**을 그대로 붙여넣기:
   ```
   https://lomjryokqskruzdhieol.supabase.co/auth/v1/callback
   ```
   ⚠️ 여기가 제일 중요해요. 오타 나면 로그인이 안 돼요. 그대로 복사-붙여넣기 하세요.
6. **만들기** 클릭.
7. 그러면 팝업으로 **클라이언트 ID** 와 **클라이언트 보안 비밀번호(Secret)** 두 개가 나와요.
   → 두 개 다 **복사해서 메모장에 붙여놓으세요.** (다음 단계에서 씁니다.)

## 2단계. 우리 서버(Supabase)에 열쇠 끼우기

방금 만든 열쇠를 우리 로그인 서버에 등록하는 단계예요.

### 2-1. 구글 로그인 켜기
1. 브라우저에서 **https://supabase.com/dashboard** 접속 → 로그인.
2. 프로젝트 목록에서 **BridgeX** (프로젝트 id: `lomjryokqskruzdhieol`) 클릭.
3. 왼쪽 메뉴 맨 아래쪽 **Authentication**(사람 모양 아이콘) 클릭.
4. 그 안에서 **Providers**(또는 Sign In / Providers) 클릭.
5. 목록에서 **Google** 을 찾아 클릭 → 오른쪽 스위치를 **켜기(ON)**.
6. 칸 두 개가 나와요:
   - **Client ID:** 1단계에서 복사한 **클라이언트 ID** 붙여넣기
   - **Client Secret:** 1단계에서 복사한 **보안 비밀번호** 붙여넣기
7. **Save**(저장) 클릭.

### 2-2. 돌아올 주소 등록하기
1. 같은 **Authentication** 안에서 **URL Configuration** 클릭.
2. **Site URL** 칸이 `https://bridgexkorea.com` 인지 확인 (아니면 그렇게 바꾸기).
3. **Redirect URLs** 에 **+ Add URL** 로 아래 두 개를 추가:
   ```
   https://bridgexkorea.com/auth/callback
   http://localhost:3000/auth/callback
   ```
   (첫 번째는 실제 사이트용, 두 번째는 테스트용이에요.)
4. **Save**(저장) 클릭.

## 3단계. 진짜 되는지 확인하기 (테스트)

1. 브라우저에서 **https://bridgexkorea.com/login** 접속.
   (혹시 몇 분 전이면 새 화면이 아직 안 떴을 수 있어요. 5분쯤 뒤 새로고침 하세요.)
2. "로그인" 버튼 아래 **또는** 줄과 흰색 **Google로 계속하기** 버튼이 보이면 성공적으로 배포된 거예요.
3. **Google로 계속하기** 클릭 → 구글 로그인 창이 뜨고, 계정을 고르면
4. **처음 가입하는 구글 계정**이면 → **회사명·전화번호·동의** 입력 화면(온보딩)이 나와요.
   입력하고 **시작하기** → 마이페이지 도착하면 완벽하게 성공! 🎉
5. **한 번 가입했던 계정**으로 다시 로그인하면 → 온보딩을 건너뛰고 바로 마이페이지로 가요.

### 잘 안 될 때 (자주 나는 문제)
- **"redirect_uri_mismatch" 에러가 떠요** → 1-3의 **A값 주소**에 오타가 있어요.
  구글 콘솔로 돌아가 리디렉션 URI를 A값과 똑같이 고치고 저장하세요.
- **버튼을 눌러도 로그인 화면으로 되돌아와요** → 2-1에서 Client ID/Secret을
  잘못 붙였거나 Google 스위치가 꺼져 있어요. 다시 확인해주세요.
- **화면에 Google 버튼이 아예 안 보여요** → 배포가 아직 반영 중이에요. 5분 뒤 새로고침.
